工厂模式

​ 定义一个创建对象的接口，让其子类自己决定实例化哪一个实体对象，工厂模式使其创建过程延迟到子类进行。

​ 大白话：首先创建一个接口，创建多个子类实现该接口。然后创建一个工厂类，通过工厂类对用户的输入进行选择，返回实体对象类赋值给新建的接口对象（多态）。

一、简单工厂模式

使用

//创建工厂

ShapeFactory shapeFactory = new ShapeFactory();

//在工厂中利用输入参数选择具体的实体对象，并将该对象返回给Shape接口

Shape shape = shapeFactory.getShape("CIRCLE");

创建过程

第一步

创建接口

public interface Shape { void draw(); }

第二步

//Rectangle类

public class Rectangle implements Shape {

@Override public void draw() { System.out.println("Inside Rectangle::draw() method."); } }

//Square类 public class Square implements Shape {

@Override public void draw() { System.out.println("Inside Square::draw() method."); } }

//Square类 public class Square implements Shape {

@Override public void draw() { System.out.println("Inside Square::draw() method."); } }

创建工厂，在工厂中根据用户的选择给定信息的实体类对象

二、抽象工厂模式

​ 抽象工厂默模式就是在简单工厂模式的基础上增加了产品族，将工厂类进行分类。

​ 对比简单工厂模式，抽象工厂模式增加了多个工厂类，分别生产不同类的产品，并将多个工厂类进行抽象得到一个超级工厂类，创建一个工厂生成类。①通过工厂生成类对用户输入进行选择，返回子类工厂类对象赋值给超级工厂（多态，超级工厂类对象指向子类对象）。②然后工厂对象根据用户输入特征返回具体的实体类对象并赋值给Shape或者Color，即完成操作。

​ 此处，也可以将Shape接口和Color接口总结为一个接口。
